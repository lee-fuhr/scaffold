# Task notes templates

Quick reference for the task notes pattern.

## Quick start

**1. Create task notes directory:**
```bash
mkdir -p /path/to/project/_notes/[task-name]
```

**2. Copy templates:**
```bash
cp /path/to/your/project/_templates/task-notes/*.md /path/to/project/_notes/[task-name]/
```

**3. Fill in context.md and start working**

## What goes where

| File | Purpose |
|------|---------|
| `context.md` | What is this task, why it matters, background, success criteria |
| `decisions.md` | Every significant decision, rationale, alternatives, trade-offs |
| `open-questions.md` | Blockers, unknowns, research needed, resolved questions |

## When to use

Create task notes when:
- Task spans multiple sessions
- Multiple agents involved
- Complex decisions with trade-offs
- High stakes or novel work
- Client work needing audit trail

## Key principle

**Update continuously as you work** - These are working memory, not documentation.

**Read completely before resuming** - Context, decisions, questions all in one place.
